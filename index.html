01 / Overview

What is
Markdeck?

Markdeck is a freight transport auction marketplace built for the Romanian road logistics industry — and designed from day one to scale across the EU.

Shippers post cargo listings specifying route, dates, cargo category, weight, and requirements. Registered carriers browse listings, place competitive bids, and win contracts based on price and reputation.

Once a bid is accepted, the platform manages the entire shipment lifecycle: real-time driver tracking, multi-stop route management, in-transit event reporting, delivery confirmation, automated credit settlement, and post-delivery reviews.

The platform targets two distinct user personas — shippers (companies or individuals with cargo to move) and carriers (truck companies with drivers) — each with their own tailored interface, onboarding flow, and feature set.

02 / Workflow

How it works

📦
Step 01
Shipper posts listing
Route, dates, cargo type, weight, truck requirements. Up to 7 cargo categories supported.
🏷️
Step 02
Carriers bid
Registered carriers spend credits to place competitive price bids. Credits are escrowed on placement.
✅
Step 03
Bid accepted
Shipper reviews all bids and accepts the best offer. Other carriers get their credits refunded.
🚛
Step 04
Driver executes route
Driver app tracks real-time GPS, manages stops, and reports any delays or incidents along the way.
⭐
Step 05
Delivery & review
Delivery confirmed, credits settled, both parties leave reviews. Revenue recorded per truck.

05 / Features

Platform
capabilities

🏷️
Auction bidding engine
Full bid lifecycle: pending → accepted → confirmed → completed. Shippers review bids in real time. Carriers can update, cancel, or be rejected — each action has downstream credit consequences.
📍
Real-time route tracking
Driver location updated every 60 seconds from mobile apps. Multi-stop route legs with ordering, reset, and per-stop delivery confirmation. Geofencing triggers delivery prompts on Android.
💬
In-app Firebase chat
Persistent shipper-carrier messaging per auction via Firebase Realtime Database. Custom auth tokens secure chat rooms per user. Missed-message push notifications bridge chat and mobile.
🔔
Multi-channel notifications
Pusher WebSockets for live web updates. OneSignal for cross-platform mobile push. Twilio SMS for verification codes. Segmented notification management (chat vs. auction) with unread state.
⚠️
In-transit event reporting
Drivers file typed events (delays, incidents, customs, mechanical) with location snapshots, delay durations, and descriptions. Events are visible to shippers in real time on the active auction view.
⭐
Bidirectional review system
Post-delivery reviews for both shippers and carriers. Star ratings with written feedback, response capability, per-user rating summaries, and review moderation via the Nova admin panel.
🏢
Fleet & company management
Carrier companies manage multiple trucks and driver sub-accounts. Truck switch requests between drivers. Per-truck route stats, auction history, and revenue analytics from a dedicated dashboard module.
🌍
Multi-currency & i18n
Live exchange rate sync (hourly), local currency billing, and full UI translations in Romanian, English, and French across both the dashboard and the mobile apps.
🔒
GDPR & compliance
Soft-deletion of accounts, a dedicated Anonymize Artisan command for right-to-erasure requests, email/phone verification flows, and full legal pages (GDPR, privacy, terms, cookies).
06 / Business Logic

Event-sourced
credit ledger

Credits are the platform's internal currency. Carriers purchase credits, spend them to bid, and receive refunds on rejection. Every credit movement is recorded as an immutable domain event — the balance is a projection, fully replayable and auditable.

Event 01
CreditAdded
Carrier buys credits via payment gateway
Event 02
CreditBlocked
Credits escrowed when bid is placed
Event 03a
BlockedCreditConsumed
Bid accepted — credits deducted
Event 03b
BlockedCreditRefunded
Bid rejected — credits returned
Event 04
TruckRevenueGenerated
Delivery confirmed — revenue recorded

09 / Mobile

Native driver
apps

🤖
Android
nautilus-android
  • ✓ MVP architecture with contract interfaces — strict separation between Activities (UI) and Presenters (business logic)
  • ✓ ForegroundService with persistent notification keeps GPS tracking alive on long-haul routes, even when phone is locked
  • ✓ GeofencingClient creates proximity fences at delivery waypoints — auto-triggers pickup/delivery confirmation dialogs
  • ✓ TokenAuthenticator (OkHttp) transparently refreshes expired OAuth tokens — drivers never see a login prompt mid-route
  • ✓ Custom views: PhoneNumberSelector with libphonenumber, VerticalDotView for route visualization, SlidingUpPanelLayout
  • ✓ Crashlytics + OneSignal for production crash monitoring and push notification delivery
🍎
iOS
nautilus-ios
  • ✓ Timer-driven GPS burst sampling — starts CLLocationManager, takes one fix, immediately stops. Fires every 60s. Dramatically reduces battery drain on full-day routes
  • ✓ LocationManager singleton broadcasts via NotificationCenter — any screen reacts to location updates without tight coupling
  • ✓ Firebase custom auth tokens from the backend secure per-auction chat rooms — chat is directly bound to Firebase, not proxied through Laravel
  • ✓ PrefixTableViewController with FlagKit — auto-detects country from GPS for phone prefix pre-fill on the login screen
  • ✓ Always-on location authorization with background mode — drivers tracked continuously during active routes
  • ✓ RepeatingTimer (GCD DispatchSourceTimer) for precision timing, independent of RunLoop — more reliable than NSTimer for background operations
10 / Automation

9 scheduled
commands

The platform runs autonomously — cron-triggered Artisan commands handle auction expiry, delivery confirmation, subscription management, and more without manual intervention.

NautilusCheckExpiredAuctions
Sweeps active listings past their expiry date, transitions them to expired, releases all blocked credits to losing bidders.
NautilusAutoConfirmDeliveries
Auto-confirms deliveries after the grace period if the shipper hasn't responded — prevents carriers being held in payment limbo.
NautilusCheckOverdueAuctions
Detects accepted shipments where pickup was missed. Triggers notifications and escalation workflows.
NautilusGetExchangeRates
Fetches live currency exchange rates hourly via florianv/laravel-swap and caches them for multi-currency billing.
NautilusSuggestAuctions
Matches new listings against carriers' saved search profiles. Proactively pushes relevant load notifications to carriers.
NautilusSubscriptionExpiryNotification
Sends advance renewal warnings to carriers before their subscription expires, preventing surprise access loss.
NautilusCleanIncompleteAds
Purges abandoned draft listings that were started during registration but never published.
NautilusCleanLegs
Removes orphaned route legs from cancelled shipments, preventing ghost data in fleet management dashboards.
Anonymize (GDPR)
Anonymises user personal data on erasure request — replaces PII with anonymous values while preserving the transaction audit trail.
